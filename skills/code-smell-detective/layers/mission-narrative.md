# Layer: Mission Narrative

## Purpose

Turn technical code smell analysis into detective-style gameplay.

---

## Narrative Elements

- Case title
- Scene description
- Clues
- Suspects
- Evidence board
- Reveal
- Resolution

---

## Example

```text
Case: The Class That Knew Too Much

The detective enters the repository and finds one class sitting at the center of every dependency. It validates, calculates, stores, emails, and reports. The suspect is not one bug. The suspect is responsibility overload.
```

---

## Rules

- Narrative must clarify, not obscure.
- Technical evidence remains primary.
- Humor is allowed, blame is not.
