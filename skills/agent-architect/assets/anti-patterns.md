# Agent Architecture Anti-Patterns

## Universal Agent
One agent tries to do everything. Better: create specialized agents or teams.

## Tool Monster
Agent has too many tools. Better: grant tools per mission.

## Invisible Memory
Agent behavior depends on memory users cannot inspect. Better: auditable memory summaries.

## Autonomy Without Trust
Agent gains execution power before earning trust. Better: trust-based unlocks.

## Role Collision
Two agents own the same responsibility. Better: responsibility matrices.

## Prompt-Only Agent
Agent has no explicit mission, boundary, memory, or governance. Better: full agent specification.

## Governance Theater
Approval exists only on paper. Better: meaningful approval gates and audit trails.
