# Agent Architect Checklists

## Agent Design Checklist

- [ ] Mission is explicit.
- [ ] Primary skill is defined.
- [ ] Supporting skills are defined.
- [ ] Role boundary is clear.
- [ ] Tool access is minimal.
- [ ] Memory scope is defined.
- [ ] Authority level is defined.
- [ ] Outputs are observable.
- [ ] Escalation triggers are defined.
- [ ] Tests exist.

## Agent Team Checklist

- [ ] Team mission is clear.
- [ ] Collaboration model is selected.
- [ ] Agent responsibilities do not overlap.
- [ ] Handoffs are defined.
- [ ] Shared artifacts are defined.
- [ ] Coordinator role is justified.
- [ ] Final output ownership is clear.
- [ ] Escalation path exists.

## Governance Checklist

- [ ] Authority is explicit.
- [ ] Approval gates are meaningful.
- [ ] Accountability is assigned.
- [ ] Tool usage is auditable.
- [ ] Memory writes are governed.
- [ ] Risky actions require approval.
- [ ] Agent can be disabled.
- [ ] Decisions can be explained.

## World Integration Checklist

- [ ] Base skills are reused.
- [ ] World adapter is defined.
- [ ] World-specific names are mapped.
- [ ] Departments are mapped.
- [ ] Missions are mapped.
- [ ] Achievements are mapped.
- [ ] Memory is scoped by world.
- [ ] Kernel changes are not required.
