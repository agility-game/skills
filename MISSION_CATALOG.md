# MISSION_CATALOG.md

Agility Game Mission Catalog

Version: 1.0

Status: Core Specification

Repository: agility-game/skills

⸻

Purpose

Missions are the primary gameplay unit within Agility Game.

A mission represents work performed by one or more skills.

Missions generate:

* Experience
* Achievements
* Reputation
* Story Progression
* Organizational Outcomes

⸻

Mission Structure

Mission
```
├── Objective
├── Participants
├── Inputs
├── Activities
├── Outputs
├── Validation
└── Rewards
```

Mission Types

Discovery Missions

Goal

Understand the problem.

Examples

* Stakeholder Interviews
* Architecture Assessment
* Threat Discovery
* Backlog Discovery

Primary Skills

* Product Owner
* Business Analyst
* Architect

⸻

Design Missions

Goal

Create a future state.

Examples

* System Design
* Threat Modeling
* Platform Design
* Agent Design

* Agent Architect

⸻

Build Missions

Goal

Mission Chains

Missions may form campaigns.

Example

Legacy Modernization Campaign

1. Assess Architecture
2. Identify Smells
3. Design Future State
4. Refactor Components
5. Validate Quality
6. Deploy Platform
7. Measure Outcomes

Campaigns tell stories inside worlds.

⸻

Dynamic Mission Generation

The Mission Engine may generate missions based on:

* World State
* Department State
* Technical Debt
* Security Risks
* Product Goals
* AI Agent Recommendations

This enables infinite replayability across worlds.
