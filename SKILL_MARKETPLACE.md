# Skill Marketplace

Version: 1.0

---

# Purpose

Defines how skills are shared and distributed.

---

# Marketplace Concept

Skills behave like installable packages.

Examples:

- architect
- developer
- product-owner
- scrum-master
- tester
- security-engineer

---

# Installation

Example:

skill install architect

skill install product-owner

---

# Dependencies

Skills may require:

- Other skills
- Capsules
- Worlds

Example:

security-architect

requires:

- architect
- security-engineer

---

# Publishing

Community members may contribute skills.

Requirements:

- Conform to specification
- Include tests
- Include scoring
- Include missions

---

# Governance

Skills are reviewed before publication.

The kernel remains protected.

Skills provide extension points.

The marketplace enables innovation.
